Lib\+TTAK is a C systems library. It does {\bfseries{not}} adopt the classic C++-\/style RAII patternâ€”where resources are acquired in a constructor and automatically released by an implicit destructor when a stack object goes out of scope. Every allocation, teardown step, and cross-\/thread transfer is observable and caller-\/driven.

\hypertarget{md_RAII_autotoc_md33}{}\doxysection{\texorpdfstring{1. Explicit Lifetimes Instead of Implicit Destructors}{1. Explicit Lifetimes Instead of Implicit Destructors}}\label{md_RAII_autotoc_md33}
{\bfseries{Traditional RAII}} relies on compiler-\/generated destructor calls tied to variable scope. The programmer does not write the release code; it happens automatically.

{\bfseries{Lib\+TTAK}} requires callers to pass a {\ttfamily lifetime\+\_\+ticks} hint at allocation time ({\ttfamily ttak\+\_\+mem\+\_\+alloc\+\_\+safe}, {\ttfamily ttak\+\_\+fastalloc}). The memory header records creation and expiration ticks. The block may become {\itshape dirty} when {\ttfamily now\+\_\+tick \texorpdfstring{$>$}{>} expires\+\_\+tick}, but it is {\bfseries{not}} freed automatically. Callers must invoke {\ttfamily ttak\+\_\+mem\+\_\+free}, {\ttfamily tt\+\_\+autoclean\+\_\+dirty\+\_\+pointers}, or an owner teardown explicitly. There are no hidden destructor calls.

\hypertarget{md_RAII_autotoc_md35}{}\doxysection{\texorpdfstring{2. Owner-\/\+Based Sandboxing ($<$tt$>$ttak\+\_\+owner\+\_\+t$<$/tt$>$)}{2. Owner-Based Sandboxing (<tt>ttak\_owner\_t</tt>)}}\label{md_RAII_autotoc_md35}
Lib\+TTAK introduces {\bfseries{owners}} as first-\/class containers for resources and functions. An owner ({\ttfamily ttak\+\_\+owner\+\_\+t}) is an explicit sandbox that\+:


\begin{DoxyItemize}
\item Registers resources and functions inside isolated maps.
\item Enforces safety policies ({\ttfamily TTAK\+\_\+\+OWNER\+\_\+\+DENY\+\_\+\+DANGEROUS\+\_\+\+MEM}, {\ttfamily TTAK\+\_\+\+OWNER\+\_\+\+STRICT\+\_\+\+ISOLATION}).
\item Requires explicit {\ttfamily \doxylink{owner_8h_a73e3bd2628d53ed3e2b168de4f999c16}{ttak\+\_\+owner\+\_\+destroy()}} to release everything inside it.
\end{DoxyItemize}

Unlike traditional RAII, where a single smart pointer owns one heap object, an owner owns {\bfseries{a graph of resources}}. Destruction of the owner releases the entire graph, but the programmer decides {\bfseries{when}} to trigger it.

\hypertarget{md_RAII_autotoc_md37}{}\doxysection{\texorpdfstring{3. Detachable Generational Arenas ($<$tt$>$ttak\+\_\+detachable\+\_\+context\+\_\+t$<$/tt$>$)}{3. Detachable Generational Arenas (<tt>ttak\_detachable\_context\_t</tt>)}}\label{md_RAII_autotoc_md37}
Lib\+TTAK allocates through {\bfseries{detachable arenas}}, not a global {\ttfamily malloc}/{\ttfamily free} pair. Each arena maintains\+:


\begin{DoxyItemize}
\item A 2D generation matrix ({\ttfamily rows\mbox{[}TTAK\+\_\+\+DETACHABLE\+\_\+\+MATRIX\+\_\+\+ROWS\mbox{]}}) that tracks active allocations.
\item A small-\/cache for chunks â‰¤ 16 bytes.
\item A quarantine row for fast-\/flipping allocations.
\item A status byte ({\ttfamily ttak\+\_\+detach\+\_\+status\+\_\+t}) that converges to {\ttfamily UNKNOWN} unless explicitly marked {\ttfamily KNOWN}.
\end{DoxyItemize}

This means memory is {\bfseries{not}} returned to the OS immediately upon {\ttfamily free}. Instead, it may be cached inside the arena or quarantined until the generation flips. Traditional RAII frees memory deterministically at scope exit; Lib\+TTAK frees it {\bfseries{deterministically through arena teardown}} or {\bfseries{lazily through epoch rotation}}, depending on the context flags.

\hypertarget{md_RAII_autotoc_md39}{}\doxysection{\texorpdfstring{4. Epoch-\/\+Based Reclamation (\+EBR) Instead of Immediate Free}{4. Epoch-Based Reclamation (EBR) Instead of Immediate Free}}\label{md_RAII_autotoc_md39}
Shared mutable state in Lib\+TTAK is protected by {\bfseries{Epoch-\/\+Based Reclamation (EBR)}}.


\begin{DoxyItemize}
\item Threads must {\bfseries{explicitly}} call {\ttfamily \doxylink{epoch_8h_a8366f378a77c7d52db1e8f02f099a88d}{ttak\+\_\+epoch\+\_\+enter()}} before accessing shared data and {\ttfamily \doxylink{epoch_8h_ac346c8d9bda60303dcac37eacdb5117e}{ttak\+\_\+epoch\+\_\+exit()}} afterward.
\item Retired pointers are deferred via {\ttfamily ttak\+\_\+epoch\+\_\+retire(ptr, cleanup)}.
\item Reclamation ({\ttfamily \doxylink{epoch_8h_aa6be6071c276481154cceb40f5720f32}{ttak\+\_\+epoch\+\_\+reclaim()}}) is deferred until all active threads have passed the current epoch.
\end{DoxyItemize}

However, Lib\+TTAK does {\bfseries{not}} force the caller to trigger every reclamation pass manually. The {\ttfamily ttak\+\_\+epoch\+\_\+gc\+\_\+t} context launches a {\bfseries{background rotate thread}} by default. This thread automatically advances epochs and performs non-\/blocking cleanup passes at adaptive intervals ({\ttfamily min\+\_\+rotate\+\_\+ns} / {\ttfamily max\+\_\+rotate\+\_\+ns}). Callers can influence its cadence through hints ({\ttfamily ttak\+\_\+epoch\+\_\+gc\+\_\+hint}) such as {\ttfamily TTAK\+\_\+\+EPOCH\+\_\+\+GC\+\_\+\+HINT\+\_\+\+ALLOC}, {\ttfamily TTAK\+\_\+\+EPOCH\+\_\+\+GC\+\_\+\+HINT\+\_\+\+IDLE}, or {\ttfamily TTAK\+\_\+\+EPOCH\+\_\+\+GC\+\_\+\+HINT\+\_\+\+COLLECT\+\_\+\+NOW}. If deterministic, caller-\/controlled timing is required, {\ttfamily \doxylink{epoch__gc_8h_a78ff9f44d1919ebb90542bfdd87f946e}{ttak\+\_\+epoch\+\_\+gc\+\_\+manual\+\_\+rotate()}} disables the auto-\/rotator so that {\ttfamily \doxylink{epoch__gc_8h_a2c9fb90ed22fe61eea9bbcafddf53625}{ttak\+\_\+epoch\+\_\+gc\+\_\+rotate()}} and {\ttfamily \doxylink{epoch_8h_aa6be6071c276481154cceb40f5720f32}{ttak\+\_\+epoch\+\_\+reclaim()}} are invoked explicitly.

This is fundamentally different from traditional RAII, where a {\ttfamily shared\+\_\+ptr} reference count drops to zero and the object is destroyed immediately. In Lib\+TTAK, even when the last logical owner releases a pointer, the physical memory may stay alive across an epoch boundary to protect concurrent readers, and the decision of {\itshape when} to advance that boundary can be either {\bfseries{automated}} or {\bfseries{manually orchestrated}}.

\hypertarget{md_RAII_autotoc_md41}{}\doxysection{\texorpdfstring{5. Signal-\/\+Aware Graceful Teardown}{5. Signal-Aware Graceful Teardown}}\label{md_RAII_autotoc_md41}
Lib\+TTAK provides signal hooks ({\ttfamily ttak\+\_\+hard\+\_\+kill\+\_\+graceful\+\_\+exit}, {\ttfamily ttak\+\_\+hard\+\_\+kill\+\_\+exit}) that interact with detachable arenas\+:


\begin{DoxyItemize}
\item {\bfseries{Graceful mode}} flushes arena caches and rows, then exits.
\item {\bfseries{Immediate mode}} exits without cleanup.
\end{DoxyItemize}

Traditional RAII runs destructors during normal stack unwinding (e.\+g., C++ exceptions), but signal handlers are not part of that unwinding model. Lib\+TTAK makes teardown {\bfseries{explicit at the signal boundary}}, ensuring arenas do not leak without logging their final state.

\hypertarget{md_RAII_autotoc_md43}{}\doxysection{\texorpdfstring{6. No Hidden Global State or TLS Caches}{6. No Hidden Global State or TLS Caches}}\label{md_RAII_autotoc_md43}
Traditional RAII frameworks (and some C allocators) rely on hidden thread-\/local caches or global free lists to amortize allocation costs. Lib\+TTAK {\bfseries{prohibits}} hidden TLS caches\+:


\begin{DoxyItemize}
\item {\ttfamily TTAK\+\_\+\+ARENA\+\_\+\+IS\+\_\+\+SINGLE\+\_\+\+THREAD}, {\ttfamily TTAK\+\_\+\+ARENA\+\_\+\+HAS\+\_\+\+OWNER}, and other flags are advertised explicitly.
\item Every thread must register with the epoch manager ({\ttfamily ttak\+\_\+epoch\+\_\+register\+\_\+thread}).
\item Arena flags, cache sizes, and quarantine limits are visible to the caller.
\end{DoxyItemize}

If a subsystem does not need EBR, it must document why. Nothing is implicit.

Lib\+TTAKâ€™s concurrency model is grounded in {\bfseries{orthogonal Latin square (OLS)}} principles\+: deterministic slot isolation minimizes contention. This is not limited to one subsystem.


\begin{DoxyItemize}

\item {\bfseries{Math\+:}} The NTT and matrix modules align work units to the same segmented shard table ({\ttfamily \doxylink{structttak__shard__table__t}{ttak\+\_\+shard\+\_\+table\+\_\+t}}) to avoid cache-\/line ping-\/pong.
\item {\bfseries{Execution\+:}} The async task pool and priority heap apply the same contention-\/minimizing slot logic to worker threads.
\end{DoxyItemize}

Extracting any of these domains into an external library would fracture the scheduling semantics. Keeping them together ensures that a network router, a math kernel, and a background GC thread all obey the same deterministic isolation rules.
