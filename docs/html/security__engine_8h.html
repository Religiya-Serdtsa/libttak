Detailed Description

Unified cryptographic dispatch engine for LibTTAK.

Provides a single entry point (ttak_security_execute) that routes encryption, hashing, and KDF requests to the best available driver (scalar, SIMD, or hardware accelerator) selected at runtime.

Supported operations: LEA, SEED, AES-256-GCM, ChaCha20-Poly1305, post-quantum signatures, fast hashing, and memory-hard KDF.

Unified I/O context passed to every cipher and hash operation.

Callers populate the relevant fields for their operation and pass this structure to ttak_security_execute(). Fields that do not apply to a given operation may be left zeroed.
